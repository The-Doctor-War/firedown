Captured Content — start fresh

The current screen treats every captured asset as a peer in a flat, chip-filtered grid. The result is a firehose: the one video you came for sits beside the page's avatars, emoji-SVGs, and four sizes of the same thumbnail. New model: prioritise, group, and collapse the noise instead of restyling it.

Today

Flat list · chips filter by type · every <img> the page touched is a peer row. You scroll a wall to find one video.

The funnel reality

Interceptor grabs all media URLs → one flat entity list → chip predicate → grid. Nothing ranks or de-dupes.

Proposed

A prioritised feed: Primary (what you came for) → More on this page → noise collapsed behind a count, opened on demand & de-duped.

What this changes in the funnel

Same interceptor, same entities — the new work is a ranking + grouping pass between the repo and the view, plus collapsible sections instead of one flat adapter:

1 · Rank

Pick the Primary: the video whose id matches the page URL, else the longest video. Cheap, deterministic.

2 · De-dupe

Fold image variants sharing an origin + path stem (mqdefault/sddefault/hq720 → one). Already have the canonical-key helper from the repo.

3 · Group + collapse

Bucket the rest by type with counts; collapse buckets that aren't the focus. The "Smart" facet is this view; the type facets still give the old flat filter when wanted.

Nothing is hidden permanently — every asset is one tap away. We just stop making you wade through avatars to reach the video. The chip rail keeps its old job as a manual facet; the default just gets smart.
