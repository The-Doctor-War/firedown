How the funnel supports it

Already there

Every entity has origin (page URL) + tabId. The repo is a per-tab FIFO. No new capture work.

Add: current-origin signal

The browser fragment already knows the active URL. Feed it to the view model so it can split entities into "origin == current" vs "everything else".

Add: group + order

Bucket the rest by origin, order by most-recent capture time, label each with domain + the group's primary item. One collapsible page card per origin.

Edge cases this handles cleanly: two YouTube videos = two origins = two page cards (never merged). Re-visiting a page re-focuses it, no dupes. A page with a primary video shows a hero; a page that's just images degrades to the grid — same rule, per page. The "All / type" chips still exist for when you want the old flat cross-session filter (e.g. "every subtitle from this whole session").
