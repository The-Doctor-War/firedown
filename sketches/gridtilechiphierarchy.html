Information hierarchy
does the visual weight match what matters?
 Flat. Three chips read as three peers, when only "what is this?" is identity. Mime carries weight; 19:34, 640×480 recede as the attributes they are. Same as B for video; CC is intentionally promoted because it gates a download choice.
Consistency with list view
same data, same hierarchy across layouts
 Diverges. List shows VIDEO · m.youtube.com · 1080p · 00:53:25 — only mime is decorated. Mirrors the list rule exactly. Toggling layouts feels like the same screen. Mostly mirrors; the CC chip is the deliberate exception (list keeps CC as primary-tinted text — same idea).
Photo competition
how much do chips fight the thumbnail?
 Heaviest. Coloured rectangles stack across the bottom edge of every tile. Lightest. One small chip + airy text; the photo leads, which is the point of grid view. Two chips + text. Still cleaner than A, slightly busier than B.
Legibility on busy art
does subtle text actually read?
 Strongest — guaranteed contrast. Fine — the scrim is already there for the title; reuses that work. White text with shadow + 86% gradient is standard YouTube/Photos treatment. Fine, same as B.
Maintenance
how many places encode the rule?
 Grid has its own rule; list has another. Two visual languages. One rule — "mime is the chip" — applied to both layouts. One-and-a-half rules; still simpler than A.

The hierarchy argument

Mime answers what is this? — it's identity, mapped to the filter chip rail at the top, and the basis of every decision the user makes on the screen. Duration and dimensions answer how big? — they're attributes of an item the viewer already classified. Equal visual weight tells the eye to compare them as peers, which is the wrong story.

The list-mode precedent

The list already commits to "only mime gets a background." Quality, duration, CC are plain text with " · " separators. If we keep grid's all-chips treatment, we have two visual languages for the same data — every change to one will require remembering to mirror it in the other.

What the scrim is for

The dark bottom gradient already exists to make the white title readable on any photo. Plain text picks up that contrast for free; backgrounded chips duplicate the work and visually fight the title above them. Reuse the substrate.

Why CC might be the one exception

Unlike duration or dimensions, CC changes what you can download (it adds a captions sibling). Treating it as an affordance — a chip — communicates "this is a thing you can act on," not just a fact about the file. If you want to make that distinction obvious at grid scale, C is honest. Otherwise B's coloured text is enough — the list does it that way.

Recommendation: B. Mime is the chip. Duration / dimensions are subtle white text with a separator. CC stays primary-tinted text (matches list). Reasoning: it picks one hierarchy — identity is decorated, attributes are not — and applies it to both layouts, so the grid and list aren't two different design systems. If the CC chip's affordance feels too quiet in testing, promote it to chip and live on C. Either way, the duration / dimensions chips should go.
